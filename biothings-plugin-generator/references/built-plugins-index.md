### signor
- **Datasource**: SIGNOR 4.0 — PhosphoSIGNOR
- **Homepage**: https://signor.uniroma2.it/PhosphoSIGNOR/
- **Target API**: pending.api
- **_id strategy**: SIGNOR interaction ID (`signor_id` column, e.g. `SIGNOR-250327`)
- **Data format**: Single TSV (PhosphoSIGNOR-specific PHP API endpoint — no `.tsv` extension on download)
- **Files ingested**: `https://signor.uniroma2.it/PhosphoSIGNOR/apis/v1/index.php?role=all&format=tsv&header=yes` (PhosphoSIGNOR-only; full SIGNOR interactome explicitly out of scope per SKILL.md §1b "specific files preferred" policy)
- **Parser pattern**: `defaultdict` groupby — merges paired PTM-view rows (enzyme→phosphosite + phosphosite→substrate) into one document
- **on_duplicates**: `error`
- **requires**: none (stdlib CSV only)
- **Output path**: `agent_outputs/signor_plugin/`
- **version.py strategy**: HEAD on SIGNOR master release file (`releases/Apr2026_release.txt`) → `Last-Modified` → `YYYYMMDD`; SHA-256 fallback on API response prefix
- **Date generated**: 2026-04-13
- **Version**: 1.1 (patched 2026-04-30)
- **Smoke test (2026-04-30)**: 13,411 unique interaction documents, 100% with `pmid` and `score` populated, no exceptions. Sample: SIGNOR-250327 (PRKACA → ABCA1@Ser2054, phosphorylation, up-regulates, score 0.503).
- **Notes**: v1.0 (2026-04-13) had a critical schema-mismatch bug — manifest's data_url pointed at the full SIGNOR interactome (27 uppercase columns) while the parser expected the 9-column lowercase PhosphoSIGNOR API format. v1.1 (2026-04-30) corrected the data_url to the PhosphoSIGNOR-specific endpoint, added version.py per SKILL.md §2b, and completed the manifest __metadata__ block. Enzyme IDs mix UniProtKB with internal SIGNOR complex (`SIGNOR-C*`) and protein-family (`SIGNOR-PF*`) IDs. version.py anchors on the SIGNOR master release file's Last-Modified, which has a version-stamped filename and will need URL update on each new monthly release; the SHA-256 fallback detects content changes regardless.

### gofcards
- **Datasource**: GoFCards — Curated Gain-of-Function Variants Database
- **Homepage**: https://www.genemed.tech/gofcards
- **Target API**: MyVariant.info
- **_id strategy**: hg19-based HGVS from `chr` + `hg19start` + `ref` + `alt` via inline `_vcf_to_hgvs()` (login-gated; `myvariant` package not assumed available)
- **Data format**: Excel (.xlsx, single sheet `total3161`)
- **Files ingested**: `gofcards_data_download.xlsx` only; `SNV.txt` excluded (hg38 coordinates — cannot join with hg19 xlsx without liftover)
- **Parser pattern**: pandas read_excel + two-pass groupby aggregation — variant-level fields (pscore, gene_symbol, etc.) separated from evidence-level list (function, disorder, pmid, animal_model, cell_model)
- **on_duplicates**: `error` (one document per unique variant after groupby)
- **requires**: `pandas`, `openpyxl`
- **Output path**: `agent_outputs/GoFCards_datasource/gofcards_plugin/`
- **version.py strategy**: Regex date parse from `http://www.genemed.tech/gofcards/` homepage; fallback to `20241018` (data file timestamp)
- **Date generated**: 2026-04-16
- **Version**: 1.0
- **Notes**: Download is login-gated — no `data_url` in manifest; files must be pre-placed in data_folder. Coordinates are hg19 (MyVariant.info natively uses hg38). SNV.txt Extreme GOF-category flags cannot be joined without liftover. CC BY-NC 4.0 license — verify commercial API consumer compatibility.

### harmonizome_achilles
- **Datasource**: Harmonizome 3.0 — Achilles Cell Line Gene Essentiality Profiles
- **Homepage**: https://maayanlab.cloud/Harmonizome/dataset/Achilles+Cell+Line+Gene+Essentiality+Profiles
- **Target API**: pending.api (gene-keyed, NCBI Entrez Gene ID primary key — joins natively to MyGene.info `entrezgene`)
- **_id strategy**: NCBI Entrez Gene ID as string (from `source_id` column of `gene_attribute_edges.txt.gz`)
- **Data format**: Three gzipped TSV files (bulk download from S3-backed static endpoint)
- **Files ingested**: `gene_attribute_edges.txt.gz`, `gene_list_terms.txt.gz`, `attribute_list_entries.txt.gz` from `https://maayanlab.cloud/static/hdfs/harmonizome/data/achilles/`
- **Parser pattern**: gzip + csv.reader, two-header-row skip, `defaultdict(list)` groupby on gene_id, attribute-metadata side-load for tissue annotations
- **on_duplicates**: `error` (one document per unique gene after groupby)
- **requires**: none (stdlib `gzip`, `csv`, `collections.defaultdict` + biothings SDK helpers)
- **Output path**: `agent_outputs/harmonizome_datasource/harmonizome_plugin/`
- **version.py strategy**: S3 `Last-Modified` HEAD on `gene_attribute_edges.txt.gz` → `YYYYMMDD`; fallback to SHA-256 hash prefix of the `script_config` endpoint
- **Date generated**: 2026-04-30
- **Version**: 1.0
- **Smoke test (2026-04-30)**: 4,831 unique gene documents yielded, 104,046 total cell-line associations, AKT1 (Entrez 207) verified with both positive and negative `weight` values and tissue annotations populated
- **Notes**: Achilles is one of 174 Harmonizome datasets — same plugin shape can be retargeted to CCLE/COSMIC/GDSC/DepMap/Replogle Perturb-seq/RummaGEO/Sci-Plex/Tahoe 100M/etc. by updating four lines (see plugin README). Multi-dataset combination requires sibling plugins (one per dataset) because all Harmonizome datasets share the same bulk-file basenames. CC BY-NC 4.0 — compatible with Su Lab posture; flag for commercial API consumer ambiguity. `weight` is binarized (`±1.0`) from continuous z-scores; switch to `gene_attribute_matrix_standardized.txt.gz` if continuous scores are needed. KG serializations (`kg_serializations.tar.gz`) and the public REST API (`/api/1.0`) are not consumed — bulk-download manifest is the canonical path per generator policy.

- **requires**: none (stdlib csv + biothings SDK helpers)
- **Output path**: `agent_outputs/rnacentral_datasource/rnacentral_plugin/`

- **Version**: 1.0
- **Notes**: Plugin ingests ID cross-references and RNA type classification only — not full disease associations or functional annotations. Gene-level entries (new in v26) are API-only, not in FTP bulk files. CC0 licensed. miRBase file is multi-species; parser filters to human (9606).

### metanetx
- **Datasource**: MetaNetX/MNXref — Metabolite ID Reconciliation Database
- **Homepage**: https://www.metanetx.org
- **Target API**: MyChem.info
- **_id strategy**: InChIKey (column 8 of `chem_prop.tsv`)
- **Data format**: Multi-file TSV (bulk download via tar.gz from FTP)
- **Files ingested**: `chem_prop.tsv` (chemical properties) and `chem_xref.tsv` (cross-references) from `mnxref_tsv.tar.gz`; other files in the archive (reac_*.tsv, comp_*.tsv) are ignored
- **Parser pattern**: Two-pass streaming — pass 1 reads chem_prop.tsv into dict keyed by MNX_ID (filtering for InChIKey), pass 2 reads chem_xref.tsv attaching grouped xrefs per compound
- **on_duplicates**: `error` (one document per unique InChIKey)
- **requires**: none (stdlib csv + biothings SDK helpers)
- **Output path**: `agent_outputs/metanetx_datasource/metanetx_plugin/`
- **version.py strategy**: Parse MNXref version from CHANGELOG + Last-Modified HEAD on tar.gz → `{version}_{YYYYMMDD}`
- **Date generated**: 2026-05-11
- **Version**: 1.0
- **Notes**: Core novel value is the cross-database metabolite ID reconciliation (ChEBI, KEGG, HMDB, BiGG, MetaCyc, LipidMAPS, SEED, Reactome, SABIO-RK, SwissLipids, enviPath, VMH). Chemical properties (formula, mass, SMILES) are included but overlap with PubChem/ChEBI. Large files (~772MB + 647MB uncompressed) require streaming parser. Reaction data and stereoisomer relationships not ingested — could be a separate plugin. CC BY 4.0 for MNXref reconciliation; individual source data carries original licenses.
